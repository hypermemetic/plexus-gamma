### Issues requiring fixes

| Issue | Affected screenshots | Fix |
|---|---|---|
| `substrate.echo.say` does not exist | `wiring-run`, `wiring-results`, `orch-wire`, `orch-run`, `orch-state` | Use a method that actually exists on the test backend, e.g. `substrate.call` |
| Replay history not seeded | `replay-list`, `replay-clear` | Seed history via `invoke.call` RPC before capturing, not via UI click |
| Assertion run produces no UI feedback | `assertion-run`, `assertion-runall` | Add a visible result badge / status to the assertion suite component |
| Edge not visible after `wiring.connectNodes` | `wiring-connect` | Possibly a canvas scale / scroll issue; increase `waitMs` or zoom to fit before capture |

## Navigation

### nav-multi-explorer.png

![Multi-explorer view](../screenshots/rpc/nav-multi-explorer.png)

The default landing view. A split-panel layout: the left sidebar shows a collapsible backend tree (substrate with 12 plugins, fidget-spinner with 1 plugin). The right panel shows detailed information about the selected `substrate` plugin — its children as clickable tags and a list of methods, with parameter input fields for the selected `call STREAM` method. Backend latency badges (`substrate 74ms`, `fidget-spinner 56ms`) appear at the top.

No nodes are on the canvas to differentiate it visually.

### wiring-run.png

![Wiring — run (error)](../screenshots/rpc/wiring-run.png)

**🐛 Error state.** Both Source and Sink nodes show red border outlines and *"error: RPC error -32603: Internal error"*. Root cause: the test uses `substrate.echo.say` which does not exist on the substrate backend (substrate exposes `substrate.call`, `substrate.hash`, etc. — not an `echo` namespace). The test passes because it only checks that `getResults` returns an array, not that execution succeeded.
